## Training intermediate checkpoints (preservation)

`adapters/nemotron_30b/{math,code,science}/checkpoint-N/` directories preserve
training history at every save step. Total ~3 GB across 3 domains. Not used by inference,
but kept for reproducibility — if a paper claim ever needs to be re-derived, the exact
checkpoint at every step is here.

If disk pressure ever forces cleanup, these are the safest deletion targets, in this order:
1. `checkpoint-N/` for N < final_step / 2 (early training, lowest reproducibility value)
2. `hidden_*/` checkpoints (emergency saves, mostly redundant)
3. `dare_sparsified/` (regeneratable from `final/` via `apply_dare_to_peft_model`)
