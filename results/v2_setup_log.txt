CRITICAL ARCHITECTURAL NOTES:
1. Backend RoutedLoRALinear (dynamic_mlx_inference.py L39) uses PER-ADAPTER weight cap:
     w = mx.minimum(routing_weight, _GLOBAL_CLAMP)
   NOT the per-layer norm-ratio clamp (gamma_l = min(1, c * ||z|| / ||m||)).
   The AdaptiveMultiLoRALinear in src/adapters/ is a REFERENCE IMPLEMENTATION
   that is NOT in the real inference path.

2. Orchestrator (orchestrator.py) returns ONE-HOT domain routing (top-1 only).
   It does NOT return a probability distribution.

3. CONSEQUENCE: The GatedRouter (src/routers/gated_router.py) cannot meaningfully
   gate K=1 vs K=2 with the current orchestrator, because it always receives
   gap=1.0 → always K=1.
